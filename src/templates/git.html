Some commands taken from the Scott Chacon video, taken from the manual or randomly googled:
  • git init
Create a new git repository
  • git clone ../dir_name/
Copy a repository
  • git checkout <random commit number>
The random commit number can be taken
  • git checkout -b <name> <another_branch>
Changes the git repository so that the named branch is taken into use, or creates a new branch in the repository if the name doesn't exist based on the current repository status or that pointed by the optional another_branch.
  • git add .
  • git commit
Commits the current added changes. This automatically brings up a file where you need to add a task description.
  • git commit -m 'task description'
Commits the current added changes using the task description. I.e. removes the git commit stage to put the comment in away from the command line.
  • git status
  • git config alias.<name> "<rule>"
  • git config --global alias.lol "log --oneline --graph --all --decorate"
so git lol is now short for git log --oneline --graph --all --decorate
  • git config --global user.name "Andrew Carter"

  • git config --global user.email "<name>@yahoo.co.uk"
so
  • git config --global color.ui true

  • git branch
Shows the repository branches and identifies the current working branch.
  • git merge <branch>
Merges a branch with the current branch. So if you create a number of branches and then merge back the desired changes back into the master.
  • git diff
  • git push
  • git fetch
